Webhooks
Webhook Events
Our webhooks send notifications for various events. Each event type follows the same payload structure.

Collection Events
Event Types
static_account.transaction.created - Triggered when a payment is received in a reserved virtual account

invoice.paid - Triggered when an invoice payment is received

Payload Structure

Field	Type	Description
event	string	The webhook event identifier
data	object	Contains the transaction details

able
Field	Type	Description
transactionId	integer	Unique identifier for the transaction
transactionReference	string	Unique reference for tracking the transaction
accountExchangeReference	string	Reference ID for the account exchange
sessionId	string	Unique session identifier
accountNumber	string	Virtual account number
accountType	string	Type of account (e.g., "STATIC")
amount	string	Transaction amount in decimal format
sourceAccountName	string	Name of the sending account
sourceAccountNumber	string	Account number of the sender
sourceBankName	string	Name of the sending bank
sourceBankCode	string	Bank code of the sending bank
destinationAccountNumber	string	Account number of the recipient
destinationAccountName	string	Name of the recipient account
destinationBankName	string	Name of the receiving bank
destinationBankCode	string	Bank code of the receiving bank
type	string	Transaction type (e.g., "INFLOW")
status	string	Transaction status: "CONFIRMED", "UNRESOLVED", or "AUTHORIZED"
narration	string	Transaction description
name	string	Account holder's name
createdAt	string	Transaction creation timestamp (ISO 8601 format)
updatedAt	string	Last update timestamp (ISO 8601 format)
metadata	object	Additional transaction metadata

const crypto = require('crypto');
function verifySignature(signature, payload, secret) {
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(JSON.stringify(payload))
    .digest('hex');
  return signature === expectedSignature;
}

Refund
Transaction refunds allow merchants to return funds to customers who have made payments through reserved accounts or invoice accounts. This feature provides a way to handle customer disputes, cancellations, or other scenarios where a payment needs to be reversed.

Overview
When a refund is initiated, the system processes the reversal of the original transaction, returning the specified amount (or full amount) back to the customer's source account. Refunds can be processed for both successful and failed transactions, depending on the specific circumstances and business rules.

Refund Process
Initiate Refund: Merchant initiates a refund request through the dashboard or API

Validation: System validates the refund request against business rules and available balance

Processing: Refund is processed and funds are returned to the customer's account

Status Update: Transaction status is updated to reflect the refund

Notification: Customer receives confirmation of the refund

Refund Types
Full Refund: Returns the entire transaction amount

Partial Refund: Returns a portion of the transaction amount

Automatic Refund: System-initiated refunds for failed or expired transactions

Manual Refund: Merchant-initiated refunds for customer service purposes

Use Cases
Customer cancellations and returns

Dispute resolution

Technical errors or system failures

Customer service adjustments

Compliance with regulatory requirements

Important Notes
Refunds can only be processed for confirmed transactions

Available balance must be sufficient to cover the refund amount

Refund processing time depends on the customer's bank and may take 1-3 business days

All refunds are logged and tracked for audit purposes

Refund policies should comply with local regulations and business terms
